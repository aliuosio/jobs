# Contract: Job Offers CSV Export Endpoint

## Endpoint
**GET** `/job-offers`

## Description
Retrieves job offers with optional format parameter. When `format=csv` is specified, returns a CSV file download containing only applied jobs.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| format | string | No | json | Response format: `json` or `csv` |
| limit | integer | No | null | Maximum number of results (JSON only) |
| offset | integer | No | null | Number of results to skip (JSON only) |

## Format: JSON (Default)

### Request
```
GET /job-offers
GET /job-offers?format=json
GET /job-offers?format=json&limit=10&offset=0
```

### Response 200 OK
```json
{
  "job_offers": [
    {
      "id": 1,
      "title": "Software Engineer",
      "url": "https://example.com/jobs/1",
      "process": {
        "job_offer_id": 1,
        "research": false,
        "research_email": false,
        "applied": true
      }
    }
  ]
}
```

## Format: CSV

### Request
```
GET /job-offers?format=csv
```

### Response 200 OK
```
HTTP/1.1 200 OK
Content-Type: text/csv; charset=utf-8
Content-Disposition: attachment; filename="applied-jobs-2026-03-22T143052.csv"

company,email,company_url,title,url,posted
"Acme Corp",hr@acme.com,https://acme.com,"Senior Developer, Backend","https://example.com/jobs/2","2026-03-20"
```

### CSV Format Details
- **Encoding**: UTF-8 with BOM (for Excel compatibility)
- **Delimiter**: Comma (`,`)
- **Quoting**: RFC 4180 compliant
  - Fields containing commas, quotes, or newlines are enclosed in double quotes
  - Double quotes within fields are escaped as `""`
- **Headers**: Always present
- **Line Ending**: CRLF (`\r\n`)

### Filter Behavior
When `format=csv`, the response includes **only jobs where `applied: true`**.

## Error Responses

### 400 Bad Request (Invalid Format)
```json
{
  "detail": "Invalid format parameter. Must be 'json' or 'csv'."
}
```

### 503 Service Unavailable
```json
{
  "detail": "Database temporarily unavailable"
}
```

### 500 Internal Server Error
```json
{
  "detail": "Internal server error"
}
```

## Example CSV Output

### Input
```sql
-- Jobs in database:
-- id=1: "Software Engineer", company="Acme Corp", applied=true
-- id=2: "Product Manager", company="Beta Inc", applied=false
-- id=3: "DevOps Engineer", company=null, applied=true
```

### Request
```
GET /job-offers?format=csv
```

### Response Body
```
company,email,company_url,title,url,posted
Acme Corp,hr@acme.com,https://acme.com,Software Engineer,https://example.com/jobs/1,2026-03-15
,,,DevOps Engineer,https://example.com/jobs/3,2026-03-18
```

Note: Job id=2 (Product Manager) is not included because `applied=false`.

## Notes
- The CSV filename is generated by the backend with current timestamp
- Filename format: `applied-jobs-{YYYY-MM-DD}T{HHMMSS}.csv`
- Empty/null fields are rendered as empty strings in CSV
