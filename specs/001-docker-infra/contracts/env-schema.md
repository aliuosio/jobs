# Environment Variable Contract

**Feature**: 001-docker-infra | **Date**: 2026-03-08

## Overview

This document defines the contract for environment variables required by the Docker infrastructure. All variables are managed via `.env` file at the repository root.

---

## Required Variables

### ZAI_API_KEY

| Property | Value |
|----------|-------|
| **Name** | `ZAI_API_KEY` |
| **Type** | string |
| **Required** | Yes |
| **Default** | (none) |
| **Secret** | Yes |
| **Consumer** | api-backend |
| **Purpose** | Authentication key for Z.ai embedding API |

**Validation Rules**:
- Must be non-empty string
- Must not contain whitespace
- Should be kept secret (never commit to VCS)

**Example**:
```bash
ZAI_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

---

## Optional Variables

### QDRANT_URL

| Property | Value |
|----------|-------|
| **Name** | `QDRANT_URL` |
| **Type** | string (URL) |
| **Required** | No |
| **Default** | `http://qdrant-db:6333` |
| **Secret** | No |
| **Consumer** | api-backend |
| **Purpose** | Connection URL for Qdrant vector database |

**Validation Rules**:
- Must be valid HTTP/HTTPS URL
- Must include port number

**Example**:
```bash
QDRANT_URL=http://qdrant-db:6333
```

---

### ZAI_BASE_URL

| Property | Value |
|----------|-------|
| **Name** | `ZAI_BASE_URL` |
| **Type** | string (URL) |
| **Required** | No |
| **Default** | `https://api.z.ai/v1` |
| **Secret** | No |
| **Consumer** | api-backend |
| **Purpose** | Base URL for Z.ai API endpoints |

**Validation Rules**:
- Must be valid HTTP/HTTPS URL
- Should not end with trailing slash

**Example**:
```bash
ZAI_BASE_URL=https://api.z.ai/v1
```

---

### QDRANT__LOG_LEVEL

| Property | Value |
|----------|-------|
| **Name** | `QDRANT__LOG_LEVEL` |
| **Type** | enum |
| **Required** | No |
| **Default** | `INFO` |
| **Secret** | No |
| **Consumer** | qdrant-db |
| **Purpose** | Logging verbosity for Qdrant |

**Allowed Values**:
- `DEBUG` - Most verbose
- `INFO` - Standard logging (default)
- `WARN` - Warnings and errors only
- `ERROR` - Errors only

**Example**:
```bash
QDRANT__LOG_LEVEL=INFO
```

## .env File Template

```bash
# ============================================
# Docker Infrastructure Environment Variables
# Generated by scripts/init-env.sh
# ============================================

# Qdrant Configuration
QDRANT_URL=http://qdrant-db:6333

# Z.ai API Configuration
# REQUIRED: Set your Z.ai API key
ZAI_API_KEY=your_api_key_here
ZAI_BASE_URL=https://api.z.ai/v1
```

## Security Requirements

1. **Never commit `.env` to version control**
   - `.gitignore` must include `.env`

2. **File permissions**
   - `.env` should have `600` permissions (owner read/write only)
   - Generated by `scripts/init-env.sh`

3. **Secret rotation**
   - `ZAI_API_KEY` should be rotated periodically
   - Update `.env` and restart api-backend container

---

## Usage in Docker Compose

Variables are referenced using `${VAR:-default}` syntax:

```yaml
services:
  api-backend:
    environment:
      - QDRANT_URL=${QDRANT_URL:-http://qdrant-db:6333}
      - ZAI_API_KEY=${ZAI_API_KEY}
      - ZAI_BASE_URL=${ZAI_BASE_URL:-https://api.z.ai/v1}
```

---

## Validation Script

The `scripts/init-env.sh` script validates:

1. `.env` file exists
2. `ZAI_API_KEY` is set (warns if placeholder)
3. URLs are properly formatted
4. File permissions are restrictive
