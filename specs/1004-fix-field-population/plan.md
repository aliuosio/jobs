# Implementation Plan: Fix Hybrid Search Field Population Bug

**Branch**: `1004-fix-field-population` | **Date**: 2026-03-25 | **Spec**: [spec.md](./spec.md)

## Summary

Restore profile chunk inclusion in `/fill-form` endpoint that was removed during LLM-based classification refactoring. The fix adds `retriever.get_profile_chunk()` call after hybrid search, inserts profile at beginning of chunks list, and attempts direct field extraction for known field types before falling back to LLM classification.

## Phase 2: Implementation Plan

### Code Changes (Single File)

In `src/api/routes.py`, function `fill_form()`, after line ~136 (chunks retrieval):

```python
# 1. Fetch profile chunk
profile_chunk = await retriever.get_profile_chunk()
if profile_chunk:
    chunks.insert(0, profile_chunk)
    chunk_count = len(chunks)

# 2. Try direct extraction for known field types
if signals:
    field_type = classify_field_type(signals)
    if field_type:
        field_value = _extract_direct_field_value(chunks, field_type)
        if field_value:
            # Early return with direct extraction result
            return AnswerResponse(
                answer=field_value,
                has_data=True,
                confidence=ConfidenceLevel.HIGH,
                context_chunks=chunk_count,
                field_value=field_value,
                field_type=field_type.value,
            )

# 3. Continue with LLM classification (profile now in context)
```

### Error Handling (FR-008)

If `get_profile_chunk()` raises exception:
- Return HTTP 503 with "Service temporarily unavailable"

### Performance (FR-007)

Target: <800ms including profile fetch
