# Unit Tests for English — Requirements Quality

Created: 2026-03-29

## Completeness
- [ ] Are the core requirement statements included for the scope? [Completeness]
- [ ] Are all identified stakeholders and roles described? [Completeness]
- [ ] Are explicit out-of-scope items documented? [Completeness]

## Clarity
- [ ] Are key terms defined (e.g., "prominent display", "visual hierarchy", "mutations")? [Clarity]
- [ ] Are acceptance criteria described with precise, measurable limits? [Clarity]

## Consistency
- [ ] Is terminology consistent across the spec (no synonyms for the same concept)? [Consistency]
- [ ] Are formatting decisions consistent with the checklist template? [Consistency]

## Acceptance Criteria Quality
- [ ] Are acceptance criteria clearly tied to each requirement? [Measurability]
- [ ] Are criteria verifiable by agent-executable tests? [Measurability]

## Scenario Coverage
- [ ] Do primary user journeys have defined acceptance criteria? [Coverage]
- [ ] Are edge cases and failure scenarios described? [Edge Case]

## Non-Functional Requirements
- [ ] Are latency or performance constraints specified? [NFR]
- [ ] Are accessibility requirements defined where applicable? [Accessibility]

## Dependencies & Assumptions
- [ ] Are external dependencies documented? [Dependencies]
- [ ] Are key assumptions explicit? [Assumptions]

## Ambiguities & Conflicts
- [ ] Are there any ambiguous terms needing quantification? [Ambiguity]
- [ ] Are there conflicting requirements that need resolution? [Conflict]

## Traceability
- [ ] Are there traceability references to spec sections for each item? [Traceability]
